# 1C AI Gateway

Шлюз искусственного интеллекта для интеграции ИИ-агентов с метаданными и данными **1С:Предприятие** через протокол **Model Context Protocol (MCP)**. 

Проект построен на базе **FastAPI**, использует **Pydantic v2** для управления конфигурацией и предоставляет асинхронного ИИ-агента (паттерн ReAct) на базе моделей **Groq (Llama 3.3)**.

---

## Статус проекта

Проект находится в активной разработке.

Уже реализовано:

- ✅ AI Gateway на Python
- ✅ MCP-клиент
- ✅ Интеграция с LLM (Groq/OpenAI)
- ✅ Базовый агентный цикл с Tool Calling
- ✅ MCP-сервер для 1С
- ✅ Исследование и описание метаданных
- ✅ Выполнение запросов через MCP

- В разработке:

- 🚧 Документация и примеры использования

⚠️ Часть, связанная с 1С, проходит тестирование. Формат инструментов и API может изменяться по мере развития проекта.

* **Python** >= 3.14

## ⚙️ Настройка окружения (`.env`)

Создайте файл `.env` в корне проекта на основе шаблона. 

> **Важно по сетевым настройкам:** Проект содержит встроенную логику автоматического определения среды. Вы можете оставить `localhost` в `.env`. При запуске внутри Docker шлюз самостоятельно переключит его на `host.docker.internal` для связи с вашей локальной машиной.

```env
ONES_BASE_URL=http://localhost:8080
GROQ_API_KEY=gsk_your_real_api_key_here
```

## 🚀 Запуск напрямую через Python (Локальная разработка)

Для локальной разработки, отладки кода и работы в PyCharm проект можно запустить напрямую в вашем виртуальном окружении.

### 1. Установка зависимостей
Убедитесь, что ваше виртуальное окружение (`venv`) активировано, и установите проект в редактируемом режиме:
```bash
pip install -e .
```
*Эта команда автоматически прочитает все зависимости из `pyproject.toml` и установит их.*

### 2. Запуск сервера Uvicorn из терминала
Находясь в корне проекта (`1c-ai-gateway/`), запустите асинхронный сервер, указав путь к переменным окружения Python через `PYTHONPATH`:

* **Для Linux / macOS / Git Bash:**
  ```bash
  uvicorn app.main:app --reload --port 8000
  ```
* **Для Windows:**
  ```powershell
  uvicorn app.main:app --reload --port 8000
  ```
> Флаг `--reload` автоматически перезапускает сервер при любом изменении кода, что очень удобно при разработке.

### 3. Быстрый запуск через скрипт (Альтернатива)
Если вы хотите запускать проект одной кнопкой внутри PyCharm без прописывания `PYTHONPATH` в терминале, добавьте в корень папки `src/app/` временный файл запуска `run.py`:

```python
import uvicorn
import sys
from pathlib import Path

# Автоматически добавляем папку src в пути поиска Python
src_path = Path(__file__).resolve().parent.parent
if str(src_path) not in sys.path:
    sys.path.insert(0, str(src_path))

if __name__ == "__main__":
    uvicorn.run("app.main:app", host="127.0.0.1", port=8000, reload=True)
```
Теперь этот файл можно запустить обычной зеленой стрелочкой в PyCharm. 
*(Рекомендуется добавить `run.py` в ваш `.gitignore`, чтобы он оставался вашим личным инструментом).*

## 🐳 Запуск в Docker

Приложение упаковано с использованием безопасной многоэтапной сборки (`multi-stage build`) и запускается от лица непривилегированного пользователя `appuser`. Встроенная переменная `PYTHONPATH=/app` автоматически управляет внутренними импортами.

### 1. Сборка Docker-образа
Обязательно указывайте точку `.` в конце команды, чтобы задать контекст сборки:
```bash
docker build -t 1c-ai-gateway .
```

### 2. Запуск контейнера
Запустите собранный образ, пробросив порт шлюза и передав файл настроек окружения:
```bash
docker run -d -p 8000:8000 --env-file .env --name ai-gateway 1c-ai-gateway
```

### 3. Полезные команды управления
* **Просмотр логов (в реальном времени):** `docker logs -f ai-gateway`
* **Остановка шлюза:** `docker stop ai-gateway`
* **Удаление контейнера:** `docker rm ai-gateway`

## 🧪 Тестирование и Покрытие

Проект полностью покрыт асинхронными тестами, изолированными от внешних сетевых запросов к 1С и Groq. Все конфигурации тестов управляются через `pyproject.toml`.

### Запуск тестов
Для корректного импорта модулей запускайте тесты из корня проекта через модуль Python:
```bash
python -m pytest
```

### Проверка покрытия кода тестами (Coverage)
```bash
# Вывод отчета в консоль
python -m pytest --cov=app

# Generation интерактивного HTML-отчета
python -m pytest --cov=app --cov-report=html
```
*После генерации детальная карта покрытия доступна в файле `htmlcov/index.html`.*
